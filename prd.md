# AI 기반 기획서 자동 생성 내부 도구

## 1. 서비스 목적 (Service Purpose)

본 서비스는 **기획자가 이미 보유한 산출물(PRD, SDD, 목업, 코드, 실행 화면 등)**을 기반으로  
**기획서(정책정의서 + 화면정의서)를 자동 생성**하는 내부 생산성 도구다.

목표는 “기획서를 대신 써주는 것”이 아니라 다음에 있다.

- 기획자가 이미 만든 산출물을 **정리·구조화**
- **기획서 초안 생성 비용 최소화**
- 기획자는 **검토·수정·의사결정에만 집중**

본 도구는 외부 공개나 상용화를 목적으로 하지 않으며,  
**조직 내부 기획 프로세스를 보조하기 위한 도구**로만 사용된다.

---

## 2. 문제 정의 (Problem Definition)

### 현재 문제

바이브코딩 확산으로 기획자가 직접 다음을 만드는 경우가 증가하고 있다.

- 목업
- MVP
- 로컬 실행 화면
- README / 간이 PRD

하지만 개발 요청 및 내부 검토를 위해서는 여전히 다음 형태의 문서가 요구된다.

- 정책정의서
- 화면정의서

### 핵심 문제

- **동일한 내용을 두 번 생산**
  - 1차: 목업 / MVP
  - 2차: 기획서
- 전 과정이 수작업
- 시간 소요 큼
- 누락 및 불일치 발생
- “왜 목업까지 만들었는데 다시 기획서를 써야 하냐”는 불만 발생

### 문제의 본질

- 기획서가 **최종 산출물**로 취급되고 있음
- 실제로는 이미 다음이 존재함
  - 의도(PRD)
  - 구조(SDD)
  - 실행 결과(목업/MVP)
- 그럼에도 이를 **재정리하는 비용**이 과도함

---

## 3. 사용자 정의 (User Definition)

- **주 사용자:** 기획자
  - 서비스/기능 기획 담당
  - 바이브코딩 또는 목업 제작 경험 보유
  - 개발 요청 또는 내부 공유용 기획서 필요

- 본 서비스는 **기획자 단독 사용**을 전제로 함
  - 개발자, 디자이너는 직접 사용 대상 아님

- **인증/권한 개념 없음**
  - 내부망 접근 및 사용 통제는 인프라 레벨에서 처리
  - 서비스 코드 내 보안/권한 로직 미구현

---

## 4. 입력 (Input)

본 서비스는 **기획자가 이미 만들어 둔 산출물**을 입력으로 사용한다.

### 지원 입력 유형

#### 문서
- Markdown 기반 PRD / SDD

#### 목업 / 화면
- Figma URL
- 이미지 파일

#### 실행 결과
- 로컬 개발 URL (localhost)
- 간단한 MVP 서비스 URL

#### 코드
- 로컬 소스 코드 (선택)

> ※ 입력은 완전성을 요구하지 않음  
> ※ 일부 입력만으로도 동작해야 함

---

## 5. 출력 (Output)

### 1차 목표 출력

- **기획서 초안**
  - 정책정의서
  - 화면정의서

- 기본 출력 포맷: **PPT**

### 현실적 제약 대응

PPT 자동 생성이 기술적으로 어렵거나 품질이 낮을 경우:

- **중간 산출물 허용**
  - HTML 기반 기획 문서
  - 구조화된 문서 (JSON / Markdown)

- 이후 **기획자가 수작업으로 PPT 변환 가능**

> 👉 “PPT 자동 생성”은 **목표**이지 **절대 조건이 아님**

---

## 6. 주요 기능 (Key Features)

### 6.1 입력 통합 및 분석

- 다양한 입력 수집
- 입력 간 중복 정보 제거
- 문서 / 화면 / 코드 기반으로 다음 추출
  - 기능
  - 정책
  - 화면 단위 구조

### 6.2 정책정의서 자동 생성

- 서비스 정책 항목화
- 조건 / 예외 / 제약 중심 구조화
- 추론으로 생성된 정책은 명확히 표시

### 6.3 화면정의서 자동 생성

- 화면 단위로 다음 구성
  - 화면명
  - 주요 UI 요소
  - 사용자 동작
  - 관련 정책 연결
- 이미지 또는 화면 캡처 포함

### 6.4 기획서 초안 생성

- **최종본이 아닌 초안**
- 기획자 수정 전제
- 구조 및 목차 표준화

---

## 7. 비기능 요구사항 (Non-functional Requirements)

### 성능

- 실시간 처리 요구 없음
- 백그라운드 처리 허용
- **30분 이내 결과 도출** 목표

### 신뢰성

- 일부 입력 분석 실패 시에도
  - 전체 프로세스 중단 없음
  - 가능한 범위 내 결과 생성

### 사용성

- `업로드 → 생성 → 결과 확인` 단순 플로우
- 중간 결과 또는 진행 상태 확인 가능

### 명확히 제외되는 범위

- 인증 / 권한 관리
- 네트워크 접근 제어
- 입력 채널 확장성 고려
- 협업, 버전관리, 히스토리 관리
- 외부 데이터 수집

---

## 8. 한계 및 전제

- 생성 결과는 **항상 기획자 검수 필요**
- AI는
  - 누락을 줄일 수는 있으나
  - 기획자의 판단을 대체하지 않음
- 본 도구는
  - “기획 자동화 도구”가 아니라
  - **기획 문서 정리 가속기**

---

## 9. 성공 기준 (KPI)

- 기획서 초안 작성 시간 **체감 단축**
  - “안 쓰는 것보다 낫다” 수준이 아님
  - “이제 이거 없으면 못 하겠다” 상태

- AI 초안 활용 비율 **70% 이상**

- 기획자의 **기획서 작성 스트레스 감소**
